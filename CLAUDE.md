# Tessera — project memory

## What this is
Tessera is a study-organization platform built around web highlighting: highlight text/images/screenshot regions on any web page, save with a deep-link anchor back to the exact spot. Snippets land in a cloud-synced library auto-grouped by source site, curatable into custom **documents** mixing material from many sites. AI study tools (summaries, grounded Q&A with citations, flashcards, quizzes) run over the user's own saved material.

- **Audience (v1):** students/self-learners; small private beta. No billing, no collaboration, Chrome-only capture.
- **Spec:** `docs/Tessera-PRD-v0.1.md` — requirements locked, founder decisions in §0 (do not re-litigate without cause). Roadmap M0–M5 in §11.
- **Standalone project.** `../luminote` is a separate, older project of the founder's (similar in-browser highlighting). **Do not use it as a reference** — founder's call: it's stale and not a good reference. Build capture/anchoring fresh.

## Architecture & stack
- TypeScript end-to-end; npm workspaces monorepo (Node ≥20).
- `packages/core` — `@tessera/core`: shared domain types, ids, anchor utils (`src/types.ts`, `ids.ts`, `anchor.ts`).
- `packages/db` — `@tessera/db`: local-first store + sync engine. `LocalStore`/`RemoteAdapter` contracts, pure LWW+tombstone reconcile (`reconcile.ts`), `SyncEngine` (outbox push + cursor delta-pull), `DexieLocalStore`, `SupabaseRemoteAdapter`, in-memory fakes. **Client-authored timestamps** (server preserves them) → "last edit wins". See `packages/db/README.md`.
- `apps/extension` (M1) — Chrome MV3 via Vite + `@crxjs/vite-plugin` + React + Tailwind. `src/content/content.ts` (selection → floating Save → `anchorFromRange` → message background; re-highlight on load via `resolveAnchor` + CSS Custom Highlight API), `src/background/service-worker.ts` (**owns the Dexie store + Supabase session**: persists captures, serves snippets-by-url, **email/password auth + `SyncEngine` cloud sync** — session in `chrome.storage` via `src/lib/supabase.ts`, claims `local` captures on sign-in; context menu, periodic alarm), `src/popup` (React auth UI + snippet list). Store lives in the background because a content script sees the *page's* IndexedDB, not the extension's. Builds to loadable unpacked `dist/`. `tsconfig` extends base (Bundler resolution, not NodeNext).
- `apps/web` (M2) — `@tessera/web`: Next.js 14 App Router + Tailwind, the synced **library platform**. A local-first SPA reusing `@tessera/core`/`@tessera/db` verbatim: browser Supabase session (`src/lib/supabase.ts` — magic-link + email/password, PKCE, `detectSessionInUrl`, localStorage; no SSR/cookie auth), Dexie reads via `dexie-react-hooks` `useLiveQuery` (`src/lib/hooks.ts`) + a `SyncEngine(['snippets'])` built on sign-in and run on load/interval/focus/online (`src/components/providers.tsx`). Routes (`src/app`): `/` browse-by-website (or flat results when searching/filtering), `/site/[domain]` page drill-down, `/snippet/[id]` detail. Pure logic (grouping/filter/sort/`#:~:text=` deep-link/format) in `src/lib/snippets.ts`; private `snippet-images` rendered via signed URLs (`src/components/library/media.tsx`). Env from the **repo-root `.env`** loaded by `next.config.mjs` (no separate web `.env`). Lean shadcn-style primitives in `src/components/ui.tsx` (not full shadcn/ui yet). `next build`/`lint`/`typecheck` green; `eslint.ignoreDuringBuilds` (root flat ESLint owns linting). Dev server: `.claude/launch.json` "web" entry (port 3001).
- `supabase/migrations/` — Postgres schema as versioned, `supabase db push`-able migrations (RLS + pgvector + FTS). `0001_init` tables: user_settings, provider_keys, snippets, tags, snippet_tags, documents, document_items, snippet_embeddings, ai_artifacts, flashcards — plus `sync_push(_table,_rows)` (conditional LWW upsert) and a `set_updated_at` trigger that honors client-supplied `updated_at`. `0002` = `snippet-images` private Storage bucket + per-user RLS.
- Local-first: IndexedDB (Dexie) mirror + sync engine, last-write-wins per record + tombstones (no CRDT field merge in v1).
- Auth: magic-link + email/password fallback. AI: provider-agnostic (Claude/GPT/Gemini), BYOK in v1. Images/screenshots copied into user storage for durability.
- Documents use a **reference model** (items point at snippets; PRD §5).

## Commands
`npm install`, then `npm run build|typecheck|lint|test` (workspace-wide) and `npm run format` (Prettier; docs ignored). `npm test` runs the `@tessera/db` Vitest suite. Build core before db (workspace order handles this).
Backend (Docker): `npx supabase start` + `npx supabase db reset` for local dev (config in `supabase/config.toml`, analytics off to stay light); `supabase link` + `db push` for hosted. `node --env-file=.env packages/db/scripts/sync-smoke.mjs` = live end-to-end sync check.
